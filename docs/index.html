@devendor/sn-cloner

The sn-cloner module provides code to create a copy of a scoped servicenow application repository that you can load back into servicenow under a new scoped namespace along side the original scoped app.

It's other noteworthy trick is merging the changes back into the original parent app repository where you can commit them and pull them back into the original application.

With some additional work, this can be part of an ICDS process for servicenow scoped application development to address one of the fundemental challenges of running a large dev effort on servicenow. That is, one instance can only run a single branch of an application on that instance at a time.

Basically the expense and complexity of having an instance per developer is prohibitive, so developers typically share a dev instance and effectively work out of the "main" branch all of the time which is managable for small teams customizing OOB functionality, but a barrier to large scale development efforts of custom applications on the platform.

Some examples are on the nested sn-cloner module page and the primary API is in the CloneUtil class.

sn-cloner

Modular cloner/merger for service-now scoped application repositories.

sn-cloner.calculateGUID(scope_name, source_sys_id)

Creates a calculated sys_id based on a namespace hash of the original sys_id.

sn-cloner.getFilesFromGlob([pathGlob])

Returns an array of filenames from expanded pathGlob.

sn-cloner.getIdsFromFiles([files])

Returns sys_ids found in files.

sn-cloner.getIdsFromGlobPath([pathGlob])

Return sys_ids found in files matching pathGlob.

sn-cloner.name

package.conf name.

sn-cloner.version

package.conf version string.

sn-cloner.CloneUtil

sn-cloner.CloneUtil(clone_scope_name[, parent_repo_path])
sn-cloner.CloneUtil#clone_app_xml_path

Path the the xml files that define the service-now app in the clone.

sn-cloner.CloneUtil#clone_path

POSIX style path to clone destination.

sn-cloner.CloneUtil#clone_scope_name

name of the scope with illegal chars removed and truncated to 18 chars for compatibility.

sn-cloner.CloneUtil#clone_scope_short_name

Just splits the x_ prefix off of this.clone_scope_name.

sn-cloner.CloneUtil#id_map_file

Path the the json file used to persist sys_id mapping for the clone so we can re-merge to parent.

sn-cloner.CloneUtil#initializeClone()

Called to initialize the clone directories and properties file on intialization when needed.

sn-cloner.CloneUtil#mergeChildToParent()

Called to merge changes from the clone back into the parent.

sn-cloner.CloneUtil#mungeParentToChild()

Called to merge changes from the clone back into the parent.

sn-cloner.CloneUtil#mungeString(string_in[, reverse])

Apply string replacements to sys_ids and scopes to change parent files to child files for "clone" or the [reverse] to change child files back into parent files for "merge".

sn-cloner.CloneUtil#parent_app_file

Path the the sys_app_%sys_id%.xml file for the parent app.

sn-cloner.CloneUtil#parent_app_xml_path

Where the sn_source_control.properties points to for the xml files.

sn-cloner.CloneUtil#parent_repo_path

POSIX style path to parent repo.

sn-cloner.CloneUtil#parent_scope_name

parsed from this.parent_app_file.

sn-cloner.CloneUtil#removeAllXml(basepath)

Called by both mergeChildToParent and mungeParentToChild to remove all desination xml files from either the destination parent or child respectively.

sn-cloner.CloneUtil#saveIdMap()

Called on init to persist the updated this.id_map to file://%this.id_map_file%.

sn-cloner.CloneUtil#updateIdMapFromChild()

Called on init to generate or update file://%this.id_map_file% and this.id_map object with corresponding clone-scoped sys_ids for each one found in this.child_app_xml_path that lacks a corresponding id for parent.

sn-cloner.CloneUtil#updateIdMapFromParent()

Called on init to generate or update file://%this.id_map_file% and this.id_map object with corresponding clone-scoped sys_ids for each one found in this.parent_app_xml_path.
